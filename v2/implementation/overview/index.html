Implementation Guide

Willkommen zum CORE Design System v2 Implementation Guide. Diese Dokumentation führt dich Schritt für Schritt durch die Integration unseres Design Systems in dein Projekt - von der Installation bis zum produktiven Einsatz.

Geschätzte Lesezeit: 15 Minuten | Setup-Zeit: 5 Minuten | Schwierigkeit: Anfänger-freundlich

🚀 Quick Start - Dein erster Component in 5 Minuten

Folge diesen 3 einfachen Schritten, um sofort loszulegen:

1 CSS einbinden

Füge das zentrale CSS in deinen HTML <head> ein:

<!-- In deinem HTML <head> --> <link rel="stylesheet" href="https://core-design-system.vercel.app/shared/master.css">

2 Component HTML kopieren

Kopiere den HTML-Code eines beliebigen Components:

<!-- Primary Button --> <button class="core-button primary"> Mein Button </button> <!-- Input mit Label --> <div class="core-input-group"> <label class="core-input-label">E-Mail</label> <input type="email" class="core-input" placeholder="mail@example.com"> </div>

3 Fertig! 🎉

Öffne deine Seite im Browser - die Components sind sofort einsatzbereit!

✅ Live Demo: Teste alle Components auf core-design-system.vercel.app

Installation & Setup

Option 1: CDN (Schnellste Methode)

Ideal für Prototypen, Demos und statische Websites:

<!-- Füge dies in deinen <head> ein --> <link rel="stylesheet" href="https://core-design-system.vercel.app/shared/master.css">
Vorteile: Keine Installation, automatisches Caching, sofort einsatzbereit
Nachteile: Keine Offline-Verfügbarkeit, keine Customization

Option 2: NPM Package (Empfohlen für Projekte)

Für Next.js, React, Vue und andere Node.js-basierte Projekte:

# Installation via NPM npm install @core/design-system # Oder via Yarn yarn add @core/design-system

Import in deinem Projekt:

// In deinem App/Layout File import '@core/design-system/master.css'
Vorteile: Version Pinning, Tree Shaking, Bundle Optimization
Ideal für: Production Apps, CI/CD Pipelines

Option 3: Manual Download

Für volle Kontrolle und Customization:

  1. Download master.css von GitHub
  2. Kopiere in dein public/css/ Verzeichnis
  3. Verlinke lokal: <link href="/css/master.css">
Vorteile: Volle Kontrolle, Offline verfügbar, anpassbar
Download: GitHub Repository →

Architecture Overview

Das Zentrale CSS System

Unser Design System basiert auf einer Single-File Architecture mit master.css als zentralem Stylesheet:

45KB CSS Bundle Size
27 Components
150+ CSS Variables
0 Dependencies

Was ist in master.css enthalten?

  • CSS Custom Properties: Farben, Typography, Spacing, Shadows
  • Design Tokens v2: Orange (#FF5A10) & Deep Blue (#08204A)
  • Component Classes: Alle 27 Components mit Variants
  • Responsive Utilities: Mobile-first Breakpoints
  • Utility Classes: Margin, Padding, Display, etc.

Semantic Versioning

Wir folgen Semantic Versioning (SemVer): MAJOR.MINOR.PATCH

v2.0.0 MAJOR: Breaking Changes (z.B. CSS Class Umbenennungen)
v2.1.0 MINOR: Neue Features (z.B. neue Components)

Was passiert bei Updates?

Customization & Theming

Brand Colors überschreiben

Passe das Design System an deine Brand an, indem du CSS Variables überschreibst:

/* In deinem custom.css NACH master.css */ :root { /* Überschreibe Primary Color */ --orange-primary: #YOUR_BRAND_COLOR; /* Überschreibe Secondary Color */ --blue-deep: #YOUR_SECONDARY_COLOR; /* Custom Gradient */ --orange-gradient: linear-gradient(135deg, #YOUR_COLOR1, #YOUR_COLOR2); }
✅ Tipp: Alle CSS Variables findest du in der Colors Documentation

Dark Mode implementieren

Beispiel für einen Dark Mode Toggle:

/* Dark Mode Variables */ :root.dark-mode { --core-bg: #1a1a1a; --core-text: #ffffff; --core-card-bg: #2a2a2a; } /* Toggle via JavaScript */ document.documentElement.classList.toggle('dark-mode');

Eigene Components erstellen

Erstelle custom Components mit CORE Tokens:

/* Nutze bestehende Variables */ .my-custom-card { background: var(--core-card-bg); border-radius: var(--radius-lg); padding: var(--spacing-xl); box-shadow: var(--shadow-md); transition: var(--transition-base); } .my-custom-card:hover { box-shadow: var(--orange-shadow-lg); transform: translateY(-2px); }

Troubleshooting & FAQ

Häufige Probleme & Lösungen

❓ Styles werden nicht geladen / Component sieht falsch aus
Lösungen:
  1. Prüfe, ob master.css korrekt eingebunden ist (Browser DevTools → Network)
  2. Stelle sicher, dass der Pfad korrekt ist (CDN vs. local)
  3. Checke Browser Console auf CSS Fehler
  4. Leere Browser Cache (Cmd/Ctrl + Shift + R)
❓ Component funktioniert nicht / JavaScript Error
Lösungen:
  1. Manche Components benötigen JavaScript - siehe Component Docs
  2. Prüfe, ob interactions.js eingebunden ist (falls benötigt)
  3. Checke Browser Console auf JavaScript Fehler
❓ CSS Konflikte mit bestehendem Styling
Lösungen:
  1. Nutze CSS Specificity: .your-wrapper .core-button { }
  2. Lade master.css VOR deinem custom CSS
  3. Verwende :where() für niedrige Specificity
  4. Erwäge CSS Modules oder Scoped Styles (React/Vue)
❓ Responsive Design funktioniert nicht auf Mobile
Lösungen:
  1. Füge Viewport Meta Tag hinzu: <meta name="viewport" content="width=device-width, initial-scale=1">
  2. Checke ob Container-Breiten responsive sind
  3. Teste mit Browser DevTools Mobile Emulation
❓ Performance Probleme / Langsame Ladezeiten
Lösungen:
  1. Nutze CDN für automatisches Caching
  2. Enable Gzip/Brotli Compression auf deinem Server
  3. Lazy Load Components, die nicht sofort sichtbar sind
  4. Verwende Production Build (minified CSS)
